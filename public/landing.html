No. Every core feature — prices, signals, Fear & Greed, Polymarket odds, exchange comparison — works with zero API keys. The only optional key is OpenRouter for the AI LLM reasoning engine, which has a generous free tier.
For each timeframe (1H, 4H, 1D), TraderPro fetches the last 50–60 candles from Binance, computes RSI(14), SMA20/50, MACD(12,26,9) and volume ratio, then scores each indicator. A weighted consensus (1D has 3× weight, 4H 2×, 1H 1×) determines the overall direction. All computation is client-side — no third-party signal service.
Browser-direct calls to CoinGecko, Alternative.me and others frequently hit 429 rate limits, especially with multiple users or tabs open. The server caches responses (e.g., Fear & Greed for 10 min, Binance tickers for 20s) and uses exponential backoff retry, so users always get data even when upstream APIs are busy.
Copy .env.example to .env, generate a strong SESSION_SECRET, set NODE_ENV=production. Then run pm2 start ecosystem.config.js --env production. PM2 will run in cluster mode across all CPU cores with automatic restart on crash and log rotation. For HTTPS, put nginx in front as a reverse proxy.
Yes. TraderPro is self-hosted — your portfolio snapshots, trade signals and historical prices are stored in a local SQLite database on your own machine. Nothing is sent to any third-party analytics service.
Yes. The dashboard is responsive and includes a PWA manifest so you can install it as an app on iOS and Android. The coin cockpit, signals and position sizer all work on small screens.
The cockpit currently tracks 10 major coins (BTC, ETH, SOL, BNB, XRP, ADA, AVAX, DOGE, DOT, LINK). Adding more is a matter of extending the COINS array in cockpit.js with the Binance pair and CoinGecko ID. A full coin search using CoinGecko's 10,000+ coin catalog is also available in the market scanner.
